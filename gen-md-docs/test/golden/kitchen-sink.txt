# Kitchen Sink API

> Version 1.0.0.

A deliberately exhaustive API used to exercise the docs generator.

**See also:** [Full documentation](https://docs.example.com)

## Servers

- `https://api.example.com/v1` — Production
- `https://staging.example.com/v1`

## Authentication

- `apiKey` — API key in header `X-API-Key` — A project API key.
- `bearerAuth` — HTTP bearer (JWT)
- `oauth2` — OAuth2 — authorization code (authorize `https://auth.example.com/authorize`, token `https://auth.example.com/token`); scopes: `pets:read` (Read pets), `pets:write` (Create and update pets)

> Reference for 7 operations, grouped by tag.

- [meta](meta/index.md) — 1 operation
- [pets](pets/index.md) — 4 operations
- [store](store/index.md) — 1 operation

# List pets

Returns all pets, optionally filtered.

`GET` `/pets`

## Query parameters

**limit** `integer` (minimum: 1, maximum: 100, default: `20`) — Maximum results

**tags** `string[]`

_deprecated, not exploded_

**filter** `object`
- **color** `string`

## Headers

**X-Request-Id** `string` uuid

## Responses

### `200` — A list of pets

[`Pet[]`](#pet)

**Headers**

- **X-Total-Count** `integer` — Total number of pets
- **X-RateLimit-Remaining** `integer`

### `400` — Bad request

[`Error`](#error)

### Pet

`object`
